# Node Garden 再構築 — 総合仕様書

## 1. 概要

### 1.1 目的

現行の Node Garden モデルをゼロベースで再構築する。ノードを単位球面上に配置し、回転運動させ、ミリタリー HUD / FUI テーマのビジュアルで描画する。

### 1.2 用語定義

| 用語         | 定義                                                           |
| ------------ | -------------------------------------------------------------- |
| ノード       | 球面上（近傍）の点。描画上の頂点                               |
| エッジ       | 2 つのノード間の接続線                                         |
| 大円         | 球の中心を通る平面と球面の交線。球面上で最大半径の円           |
| 小円         | 球の中心を通らない平面と球面の交線。緯度線が典型例             |
| 測地線       | 球面上の 2 点間の最短経路（大円の弧）                          |
| FUI          | Fantasy User Interface。SF 作品に登場する架空の UI             |
| HUD          | Head-Up Display                                                |
| TSL          | Three.js Shading Language。WebGPURenderer 向けシェーダ記述言語 |
| ε オフセット | ノードの球面からの微小距離ずらし。半径を `1 ± ε` とする        |

### 1.3 背景・問題

現行実装は次の構成を取る。

- 立方体内ランダム分布
- 直線運動 + 壁反射
- 距離閾値エッジ（全ペア走査）
- 汎用マテリアル（テーマ性なし）

球面幾何に基づく配置 + 運動モデルへ移行し、ミリタリー HUD の統一ビジュアルを導入する。

### 1.4 期待効果

| 指標               | 現状             | 目標                                              |
| ------------------ | ---------------- | ------------------------------------------------- |
| ノード配置         | 立方体内ランダム | 単位球面上（± ε）                                 |
| 運動モデル         | 直線 + 壁反射    | 球面上回転（大円 / 小円）                         |
| エッジアルゴリズム | 距離閾値のみ     | 4 種（距離, k-NN, MST, Gabriel）※ Delaunay は断念 |
| ビジュアル         | テーマなし       | ミリタリー HUD / FUI 統一テーマ                   |
| GUI 操作レスポンス | 未計測           | 500 ms 以内                                       |

### 1.5 デザインキーワード

- military HUD UI design
- tactical interface blue grid UI
- futuristic HUD blue minimal
- FUI military interface
- radar UI blue theme

## 3. 共通設計方針

- **描画 / 計算の分離**: `simulation.ts` は Three.js 非依存の純粋計算。描画は `index.ts` が担当
- **関数型スタイル**: class 利用を最小化し、状態オブジェクト + 純粋関数で構成する
- **エッジ戦略パターン**: アルゴリズム切替は関数の差し替えで実現する（Phase 2 で導入）
- **パラメータ集約**: 全パラメータを `params.ts` に集約し、GUI バインド可能な形で定義する
- **WebGPU 優先**: レンダラは `WebGPURenderer`、シェーダは TSL で記述する
- **ノード上限**: 最大 100（球面上の密度を考慮）

## 4. 運動モデルの補足

ヒアリングで「回転中心が球の中心と一致しない」オプションについて議論があった。結論を整理する。

- 球面拘束を維持する場合、回転軸は球の中心を通る必要がある（通らなければノードが球面を離脱する）
- 回転軸が球の中心を通る条件下での軌道バリエーション:
  - **大円**: 回転軸 ⊥ 初期位置ベクトル → 球面上の最大円を周回
  - **小円**: 回転軸と初期位置ベクトルが斜め → 緯度線状の小さな円を周回
  - 軌道の大きさは回転軸と初期位置の角度で自然に決まる
- 楕円軌道について: 単一回転では球面上に楕円は生じない。楕円的な経路が必要な場合、2 軸回転の合成（歳差運動）で実現できる。Phase 4 以降の拡張候補として記録する
