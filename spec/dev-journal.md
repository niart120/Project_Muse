# Dev Journal

実装中の設計上の気づき・疑問・バックログ送りタスクの記録。

## 2026-04-10: カメラ操作パラメータの GUI 公開

### 現状

OrbitControls は `src/core/controls.ts` で初期化されているが、ダンピング・ズーム範囲・回転速度などのパラメータは固定値でハードコードされている。

### 観察

視覚調整中にカメラの距離や感度をランタイムで変えたい場面がある。Phase 3 (ビジュアルポリッシュ) のスコープではなく、操作性改善として独立した対応が適切。

### 方針

Phase 3 完了後にバックログとして対応する。`controls.ts` のパラメータを `NodeGardenParams` またはコア共通パラメータに昇格させ、GUI から操作可能にする。

## 2026-04-10: 速度パラメータのデフォルト値調整

### 現状

`speedMultiplier`, `angularSpeedMin`, `angularSpeedMax` のデフォルト値は初期実装時に仮決めした値のまま (`1.0`, `0.1`, `0.5`)。

### 観察

実際にビジュアルを確認しながらデフォルト値を追い込む作業が未実施。ノード数やエッジアルゴリズムとの組み合わせによって「ちょうどいい速度」が変わるため、プリセットごとに速度感も含めたい可能性がある。

### 方針

Phase 3 完了後、実際の描画を見ながらデフォルト値を調整する。カラープリセットに速度パラメータを含めるかは別途検討。

## 2026-04-10: パラメータ設定の localStorage 永続化

### 現状

GUI で変更したパラメータはページリロードで `defaultParams` にリセットされる。

### 観察

調整したパラメータを維持したいという要求は自然だが、永続化にはシリアライズ・バージョニング (パラメータ追加時の互換性)・リセット機能の設計が必要になる。Phase 3 のスコープ外。

### 方針

独立した機能として Phase 4 以降で対応する。`JSON.stringify` / `JSON.parse` + localStorage が最小構成。パラメータ構造が安定してから実装した方がバージョニングの手間が減る。

## 2026-04-10: パラメータリセット機能の追加

### 現状

GUI にはリセットボタンがなく、デフォルト値に戻すにはページをリロードする必要がある。

### 観察

localStorage 永続化とセットで実装するのが自然。リセット対象の粒度 (全体リセット / フォルダ単位 / 個別パラメータ) を決める必要がある。

### 方針

localStorage 永続化と同時に対応する。lil-gui にはフォルダ単位の `reset()` API があるため、それを活用する方向で検討。
